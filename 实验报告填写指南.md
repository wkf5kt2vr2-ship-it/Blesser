```python
import pandas as pd

# 1. 读取原始数据
input_path = "./Answer-Sheet-Data-Truth/raw_data.csv"
df = pd.read_csv(input_path)
print("原始数据量:", len(df))
print(df.head())

# 2. 清洗：去重、去空、统一格式
df = df.drop_duplicates()
df = df.dropna(subset=["question", "answer"])
df["question"] = df["question"].astype(str).str.strip()
df["answer"] = df["answer"].astype(str).str.strip()

# 3. 可选：异常值过滤（示例）
df = df[df["question"].str.len() > 0]
df = df[df["answer"].str.len() > 0]

# 4. 保存清洗结果
output_path = "./Answer-Sheet-Data-Truth/clean_data.csv"
df.to_csv(output_path, index=False, encoding="utf-8-sig")

print("清洗后数据量:", len(df))
print("已保存:", output_path)
```

## 4）【总结】整理数据集方式的优点与不足（可直接引用）

### 优点

1. **流程标准化**：清洗、标注、扩增、划分步骤明确，便于复现实验。  
2. **数据质量提升**：去重、去空、格式统一后，噪声更少。  
3. **样本覆盖增强**：通过扩增提高数据多样性，提升模型泛化能力。  
4. **训练评估规范**：训练/验证集拆分便于客观评估模型效果。  
5. **可追溯性较强**：每步保留中间文件和截图，便于审查与复盘。

### 不足

1. **人工成本高**：逐页标注与截图耗时，效率依赖人工。  
2. **标注一致性风险**：多人标注时可能存在标准不统一问题。  
3. **扩增可能引入噪声**：不合理扩增会偏离真实分布，反而影响模型。  
4. **数据泄露风险**：若报告包含真实密钥/隐私信息，存在安全隐患。  
5. **代表性受限**：若原始数据场景覆盖不足，模型上线后可能出现偏差。

### 改进建议

- 建立统一标注规范与抽检机制；
- 对扩增策略做质量评估（人工抽样 + 自动规则）；
- 报告脱敏（密钥、账号、个人隐私全部打码）；
- 引入自动化流水线（脚本化清洗、自动统计、自动可视化导出）。
